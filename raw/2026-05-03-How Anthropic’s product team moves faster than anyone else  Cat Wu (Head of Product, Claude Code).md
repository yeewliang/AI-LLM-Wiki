In this episode of Lenny's Podcast, Cat Wu (Head of Product for Claude Code at Anthropic) discusses how Anthropic builds products, the shifting landscape for Product Managers, and how to maximize leverage using AI tools.

Here are the key takeaways from the conversation:

**The Evolving Role of the AI Product Manager**

- **Shrinking Timelines:** Cat explains that because AI accelerates engineering and model capabilities improve so rapidly, product timelines have dropped from 6-12 months to just a few weeks or even a single day \[00:05:08\].
- **Blurring Lines:** The boundaries between engineering, design, and product management are dissolving. To succeed, PMs need strong "product taste" and the ability to accurately gauge the technical difficulty of a feature \[00:16:18\].
- **Evaluating Models:** Crafting "evals" and spending time testing the model’s boundaries to discover what causes errors are critical emerging skills for modern PMs \[00:54:32\].

**Anthropic's Speed and Culture**

- **Process vs. Speed:** Anthropic minimizes bureaucratic friction. By defining clear goals and leaning heavily on team principles, engineers are empowered to independently ship features without being blocked by cross-functional alignment \[00:06:59\].
- **Addressing the Leak:** Cat discusses the recent Claude Code source code leak, clarifying that it was a human error involving a human collaborating with Claude to write a PR, and that they have since hardened their internal safeguards \[00:12:05\].

**When to Use Claude Code vs. Co-Work**

- **Claude Code (CLI & Desktop):** Cat recommends using Claude Code primarily when the output is software code. The CLI is best for rapid tasks, while the Desktop app is ideal for front-end development because it allows you to preview web apps visually \[00:32:46\].
- **Co-Work for Non-Code Tasks:** Co-Work is designed for workflows like drafting documents, reaching inbox zero, or creating presentations. By securely integrating Co-Work with external data sources like Slack, Gmail, and Google Drive, the model can digest scattered information to autonomously generate polished assets \[00:36:10\].

**Designing for Smarter Models**

- **Removing Crutches:** As new frontier models are released, Anthropic frequently strips away UI features that previously acted as a crutch. For example, older models required a structured "to-do list" UI to complete a large refactor, whereas newer models natively hold the context to finish the job \[01:01:09\].
- **Unlocking New Features:** Advanced models eventually cross accuracy thresholds that make entirely new products possible, such as fully autonomous code-review agents that reliably block bugs before a code merge \[01:03:43\].

**Career Leverage in the AI Era**

- **Automate the Tedious:** Cat advises professionals to hand off the tedious, repetitive parts of their jobs to AI agents so they can redirect their focus toward high-value, creative problem-solving \[01:07:54\].
- **Push for 100% Accuracy:** Many people build workflows that get to 95% accuracy and stop. Cat pushes users to apply the elbow grease necessary to refine prompts and give models feedback until an automation hits 100% reliability, as that is when true leverage is gained \[01:09:53\].
